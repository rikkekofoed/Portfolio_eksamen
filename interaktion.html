Interaktionsudvikling

HTML

HTML står for Hyper Text Markup Language og det er det "sprog" man skriver hjemmesider i, når man sætter den basale struktur op.

Det er i HTML at man bygger en hjemmesides skelet op med f.eks. navigationsbar, header, footer, overskrifter og sections.

HTML består overordnet set af to dele: en head og en body. I head'en (ikke at forveksle med en header) placeres alt det man ikke ser når man besøger en webside.
Det gælder f.eks. metatags og links til stylesheets osv.
I body'en placeres alt indholdet så, altså alt hvad man umiddelbart ser på selve hjemmesiden, skal placeres her. Det gælder for alt tekst, billeder og f.eks. også en navbar.

CSS

CSS står for Cascading Style Sheets og det er det "sprog" man bruger til at style sin HTML og altså bestemme hvordan skelettet ser ud.

Hvis man tænker på HTML som skelettet på et menneske, vil CSS være huden på mennesket. Det er i CSS man bestemmer skrifttypen, farverne, skriftstørrelsen og meget mere.
Det er også i CSS at man kan positionere sit indhold, f.eks. i form af Flexbox eller CSS Grids.
Den måde vi primært har arbejdet med positionering af indhold er gennem CSS Grids, som jeg kommer ind på senere.

Versionsstyring

Man bruger versionsstyring til at sikre, at man altid kan gå tilbage til en anden version, hvis der skulle ske en fejl med det projekt man er i gang med.
Det kan være meget simpelt at versionsstyre ved f.eks. navngivning af mapper og filer med dertilhørende noter omkring ændringerne. Man kan altså gøre det manuelt.
Langt de fleste vælger dog at bruge et Version Control System (VCS), hvor der er en software der automatisk går ind og holder styr på forskellene mellem versioner.
En af de meget udbredte systemer, og den vi er blevet introduceret til i undervisningen, er Git-protekollen, der altså er et versionsstyrings system.
Det er muligt selv at oprette en Git-server, men man kan også sagtens bruge en online tjeneste som f.eks. Github.
Github er den tjeneste vi er blevet introduceret til, og som vi bruger til f.eks. vores projekter.

Github

GitHub fungerer som en online server til versionsstyring med Git - så behøver man ikke selv oprette en.
GitHub er blevet meget udbredt blandt webudviklere, og bruges i dag også som en slags portfolio for ens kode.
Github kan bruges via kommandolinjen, eller som langt de fleste vælger via GitHub Desktop.

GitHub fungerer ved, at man opretter et repository som er en slags projektmappe, hvori alle ens filer for projektet ligger.
Efterhånden som man får lavet noget på sit projekt committer man, og det fungerer ligesom hvis man gemmer en fil på traditionel vis
med den funktionalitet at man kan gå tilbage til tidligere versioner.
Det kan også være en god idé at lave en ny branch som er en slags parallel version af ens master branch som er den branch der gælder.
På den nye branch kan man altså lave ændringer, uden at de berører ens master.
Når man er ved at være ved vejs ende kan man lave en pull request hvor man egentlig vil have forslag til en ny version, med rettelser og alt det der.
Der kan man også se hvilke diffs der er, hvilken betyder hvilke forskelle der er mellem to commits.
Når man er tilfreds med det kan man merge det hele sammen, og det betyder at man samler ændringer i én branch med en anden branch.

GitHub gør det også ret nemt at arbejde flere sammen om et projekt, netop på grund af den måde den versionsstyrer på.
Vi bruger som sagt GitHub når vi laver projekter, hvilket vil sige, at koden til denne portfolio også er at finde på GitHub.
